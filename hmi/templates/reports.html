{# ============================================================================ reports.html — TLS-350 Report Viewer ============================================================================ This template renders the REPORTS page of the TLS-350 HMI (Human-Machine Interface). It extends "base.html" to inherit the common site layout (header, navigation sidebar, footer, stylesheets, etc.). Key features: 1. Tabbed navigation — A horizontal row of links lets the operator switch between five report categories: DELIVERY, LEAK DETECT, DIAGNOSTICS, SENSORS, and TANK CONFIG. The active tab is highlighted via the Jinja2 conditional that adds the "active" CSS class when report_type matches. 2. Report output — The report text for the currently selected tab is rendered inside a styled terminal-style block. The variable "report_text" is populated server-side by querying the ATG (Automatic Tank Gauge) backend with the appropriate TLS command for the chosen report type. 3. Connection failure warning — If the backend could not reach the ATG device (i.e. the "connected" flag is False), a red-styled warning banner is displayed to alert the operator that ATG communication has failed. Template variables expected from the server: - report_type (str) : One of 'delivery', 'leak', 'diagnostics', 'sensors', or 'config'. Determines which tab appears active. - report_text (str) : Pre-formatted text output of the selected report, ready for display in the terminal block. - connected (bool) : Whether the HMI successfully connected to the ATG device. When False, the failure banner shows. ============================================================================ #} {% extends "base.html" %} {# --- Page title (appears in the browser tab via base.html's block) --- #} {% block title %}TLS-350 - REPORTS{% endblock %} {% block content %} {# --- Page heading displayed at the top of the content area --- #} <div class="page-title">REPORTS</div> <!-- ======================================================================== Tabbed Report Navigation ======================================================================== Each tab is a plain anchor (<a>) that navigates to the same /reports endpoint but with a different "type" query parameter.
